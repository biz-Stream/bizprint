# start-task スキル

## Purpose

リポジトリに変更を加える前に、GitHub イシューとブランチを作成する。
main ブランチでの直接作業は hook でブロックされるため、このスキルを必ず最初に実行する。

## Usage

```
/start-task <タスクの説明>
```

## 手順

### 1. イシュー作成

ユーザーからタイトルと説明を確認してから作成する。

```powershell
gh issue create --title "<タイトル>" --body "<説明>"
```

### 2. ブランチ作成

イシューの内容に応じてプレフィックスを選択:

| プレフィックス | 用途 |
|---|---|
| `feature/` | 新機能追加 |
| `bugfix/` | バグ修正 |
| `hotfix/` | 緊急修正 |
| `chore/` | 設定変更・リファクタリング等 |
| `docs/` | ドキュメントのみの変更 |

```powershell
git checkout main
git pull origin main
git checkout -b <プレフィックス>/<イシュー番号>
```

### 3. 作業開始

コード修正前にユーザーに修正方針を確認する。

## 完了条件

- GitHub イシューが作成されている
- 作業用ブランチが作成され、チェックアウトされている

## MUST

- イシュー作成前にユーザーからタイトルと説明の確認を得ること。
- main での直接作業は禁止（hook でブロックされる）。

## MUST NOT

- ユーザーの確認なしにイシューを作成しないこと。

## エラー対応

| エラー | 対処 |
|---|---|
| 同名ブランチが既に存在する | `git branch -a` で確認し、既存ブランチをチェックアウトするか別名にする |
| main が最新でない | `git pull origin main` で最新化してからブランチ作成 |
| gh 認証エラー | `gh auth status` で認証状態を確認 |
