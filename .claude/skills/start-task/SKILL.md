# start-task スキル

開発タスクを開始するためのワークフロー。

## 手順

### 1. イシュー作成

```bash
gh issue create --title "<タイトル>" --body "<説明>"
```

- ユーザーからタイトルと説明を確認してから作成する。

### 2. ブランチ作成

イシューの内容に応じてプレフィックスを選択:

| プレフィックス | 用途 |
|---|---|
| `feature/` | 新機能追加 |
| `bugfix/` | バグ修正 |
| `hotfix/` | 緊急修正 |
| `chore/` | 設定変更・リファクタリング等 |

```bash
git checkout main
git pull origin main
git checkout -b <プレフィックス><イシュー番号>
```

### 3. 作業開始

- `main` での直接作業は**禁止**。必ずブランチで作業する。
- コード修正前にユーザーに修正方針を確認する。
