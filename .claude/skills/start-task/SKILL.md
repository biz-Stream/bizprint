# start-task スキル

## Purpose

リポジトリへのあらゆる変更の前に、GitHub イシュー作成とブランチ作成を行う。
別セッションで作業中の場合はワークツリーで分離し、ブランチ競合を防ぐ。

## Usage

```
/start-task <タスクの説明>
```

## 手順

### 0. 不要なワークツリーのクリーンアップ

`git worktree list` で `.claude/worktrees/` 配下にワークツリーが残っていないか確認する。

残っている場合はユーザーに一覧を提示し、削除するか確認する:

> 以下のワークツリーが残っています。不要なものがあれば削除しますか？
> - `.claude/worktrees/chore-123` (ブランチ: chore/123)

ユーザーが削除を希望した場合:

```powershell
git worktree remove .claude/worktrees/<name>
```

削除に失敗した場合（未コミット変更が残っている等）はユーザーに状況を伝え、`--force` で強制削除するか判断を委ねる:

```powershell
git worktree remove --force .claude/worktrees/<name>
```

ブランチも不要であればユーザーに確認の上で削除する:

```powershell
git branch -d <ブランチ名>
```

`-d` がマージ未済で失敗した場合はユーザーに伝え、強制削除 (`-D`) するか判断を委ねる:

```powershell
git branch -D <ブランチ名>
```

リモートにもブランチが残っている場合はユーザーに確認の上で削除する:

```powershell
git push origin --delete <ブランチ名>
```

残っていない場合、またはクリーンアップ完了後、手順 1 へ進む。

### 1. イシュー作成

ユーザーからタイトルと説明を確認してから作成する。

```powershell
gh issue create --title "<タイトル>" --body "<説明>"
```

### 2. ブランチ作成の確認

CLAUDE.md の「Main branch」設定（gitStatus に記載）からデフォルトブランチを取得する。
イシューの内容に応じてプレフィックスを選択し、ユーザーに確認する:

| プレフィックス | 用途 |
|---|---|
| `feature/` | 新機能追加 |
| `bugfix/` | バグ修正 |
| `hotfix/` | 緊急修正 |
| `chore/` | 設定変更・リファクタリング等 |
| `docs/` | ドキュメントのみの変更 |

> イシュー #XX を作成しました。`<デフォルトブランチ>` から `<プレフィックス><イシュー番号>` ブランチを作成します。よろしいですか？

- ユーザーが了承 → 手順 3 へ
- ユーザーが派生元やプレフィックスの変更を希望 → 修正して手順 3 へ
- ユーザーが現在のブランチのまま作業を希望 → ブランチ作成をスキップして作業開始

### 3. ブランチ作成

#### 3a. 同時作業の検知とワークツリー分離

現在のブランチが派生元ブランチと異なる場合、別セッションでの作業中の可能性を検知する。

**判定フロー:**

```
現在のブランチが派生元と異なる？
├─ No → 3b. 通常のブランチ作成へ
└─ Yes → 未コミット変更 or 未プッシュコミット or 上流ブランチ未設定？
     ├─ Yes → 「別作業が進行中の可能性があります」
     │         → A. ワークツリーで分離（推奨）/ B. そのまま切替
     └─ No  → ユーザーに確認
               「現在 `<ブランチ名>` にいます。このブランチの作業は完了していますか？」
               ├─ 完了済み → 3b. 通常のブランチ作成へ
               └─ まだ（PR未作成等）→ ワークツリーで分離
```

**検知コマンド:**

```powershell
# 未コミット変更の確認
git status --porcelain

# 未プッシュコミットの確認（上流ブランチがある場合）
# まず上流ブランチの存在を確認してからログを取得する
# 上流ブランチが存在しない場合（新規ブランチで未push）は「未プッシュあり」と同等に扱う
# PowerShell の && は左辺が失敗すると右辺を実行しないため、上流未設定時は安全にスキップされる
git rev-parse --abbrev-ref @{u} 2>$null && git log @{u}..HEAD --oneline
```

**PR未作成の確認（クリーン状態でユーザーに確認する際の補助情報）:**

```powershell
# 現在のブランチに対するオープンなPRがあるか確認
gh pr list --head <現在のブランチ名> --state open
```

PR が存在しない場合は「PR未作成の可能性があります」とユーザーに補足する。

**ワークツリーで分離する場合:**

1. `EnterWorktree` ツールでワークツリーを作成する（name はブランチ名のスラッシュをハイフンに置換。例: `chore/128` → `chore-128`）
2. ワークツリー内で派生元からブランチを作成:
   ```powershell
   git checkout -B <プレフィックス><イシュー番号> origin/<派生元ブランチ>
   ```

#### 3b. 通常のブランチ作成（ワークツリー不要の場合）

```powershell
git checkout <派生元ブランチ> && git pull origin <派生元ブランチ> && git checkout -b <プレフィックス><イシュー番号>
```

### 4. 作業開始

コード修正前にユーザーに修正方針を確認する。

## MUST

- リポジトリへのあらゆる変更の前にイシューとブランチを作成すること
- ユーザーが「着手して」「やって」等と指示した場合でも、イシュー・ブランチ未作成なら先に作成する
- イシュー作成前にユーザーからタイトルと説明の確認を得ること

## MUST NOT

- main ブランチで直接作業してはいけない（hook でブロックされる）
- ユーザーの確認なしにイシューを作成してはいけない

## 完了条件

- GitHub イシューが作成され、作業用ブランチにチェックアウトした状態（通常 or ワークツリー内）

## エラー対応

| エラー | 対処 |
|---|---|
| 同名ブランチが既に存在する | `git branch -a` で確認し、既存ブランチをチェックアウトするか別名にする |
| main が最新でない | `git pull origin main` で最新化してからブランチ作成 |
| gh 認証エラー | `gh auth status` で認証状態を確認 |
